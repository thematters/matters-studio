### 短期 workaround

每次 squash merge 後，到 CF dashboard → Workers & Pages → matters-studio → **Deployments** → 三點選單 → **Create deployment** 手動觸發。

### 永久修法（5 步驟，2 分鐘）

1. CF dashboard → **Workers & Pages → matters-studio → Settings → Builds**
2. 找「Git Repository」區塊 → 點 **Disconnect** → 確認彈窗
3. 同個位置變回 **Connect to Git** → 點 → 重連 `thematters/matters-studio`
4. GitHub App 權限頁要授權的話按 **Allow**（給 thematters org）
5. Production branch 確認是 `main`，Build / Deploy command 會從 `apps/web/wrangler.toml` 自動讀

接好之後最新 commit `7d16bae` 應該會自動進 build queue。

### 驗證 build 已生效

```bash
# 抓 live bundle hash
curl -s https://matters-studio.mashbean-581.workers.dev/ | grep -oE 'assets/index-[A-Za-z0-9_-]+\.js'

# 確認 6 張底圖都 bundle 進去
```

## 3. NPM publish — 給技術同事的詢問範本

`design-system` repo 的 Phase 5 寫好了 changesets workflow，理論上 merge version PR 就會自動 publish 到 npm。但：

- npm 上 `@matters` org 還沒給 mashbean 加 publisher 權限
- design-system repo 的 GitHub Actions 還沒設 `NPM_TOKEN` secret

兩個都需要技術同事（或有 npm @matters owner 權限的人）動手。**直接傳這段給 tech 同事即可**：

---

> 嗨，要請你幫忙設 design-system 的 npm publish，兩件事：
>
> **1. npm @matters org 加我（mashbean）為 member**
>
> 我的 npm username 是 `mashbean`（如有不同我再給你正確的 handle）。
>
> 麻煩到 <https://www.npmjs.com/settings/matters/members> 加我為 **Member**（不需要 Owner）。
> Member 權限就足以 publish 到 `@matters/*` 命名空間。
>
> **2. design-system repo 設 `NPM_TOKEN` secret**
>
> Repo: <https://github.com/thematters/design-system>
>
> 步驟：
>
> 1. 用一個有 `@matters` org publish 權限的 npm 帳號（你或新 bot 帳號都可以）登 npmjs.com
> 2. 到 Account Settings → Access Tokens → **Generate New Token** → 選 **Granular Access Token**
>    - Name: `matters-design-system-ci`
>    - Expiration: 1 year
>    - Permissions: **Read and write** to `@matters/*` packages（其他都不勾）
> 3. 複製 token（`npm_xxx...`）
> 4. GitHub repo → Settings → Secrets and variables → **Actions** → **New repository secret**
>    - Name: `NPM_TOKEN`
>    - Secret: 貼 token
>
> 設好之後，下一次 changesets bot 開的 「Version Packages」 PR 被 merge 時，
> CI 就會自動 publish `@matters/design-system-tokens` 和 `@matters/design-system-react` 到 npm。
>
> 相關文件 [`docs/releasing.md`](https://github.com/thematters/design-system/blob/main/docs/releasing.md) 已寫好背景。

---

設好之後，回到我這邊：

```bash
# 如果有，trigger Release workflow

# 或者直接 push commit 到 main（changesets bot 會開 Version Packages PR），
```
