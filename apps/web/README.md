# `@matters-studio/web`

Vite + React 19 frontend for Matters Studio.

## Scripts

```bash
pnpm dev          # http://localhost:5173
pnpm build        # tsc + vite build → dist/
pnpm preview      # serve the built bundle
pnpm typecheck    # tsc --noEmit
```

## Env vars

`apps/web/.env.local`:

```
VITE_API_BASE_URL=http://localhost:8787      # Worker URL
VITE_RENDER_FALLBACK=1                       # 1 = rasterize the preview iframe in-browser via html-to-image
                                             # 0 = call the Worker's /render-image (proxies to services/render)
```

## Routes (TanStack file-based)

- `/` — Dashboard (3 tiles, only `/og-image` is enabled in Phase 9.1)
- `/og-image` — OG Image wizard

`routeTree.gen.ts` is generated by `@tanstack/router-plugin` at dev/build
time and is git-ignored.

## Vendoring

The frontend does **not** depend on the design-system npm packages
(which are not published publicly yet). Instead, four DS components and
a handful of template assets are vendored into this repo:

```
apps/web/src/components/Button/
apps/web/src/components/TextField/
apps/web/src/components/Dialog/
apps/web/src/components/Toast/
apps/web/src/styles/tokens.css
apps/web/src/og-template/template.html
apps/web/src/og-template/shared/template-base.css
apps/web/src/og-template/shared/tokens.css
apps/web/src/og-template/shared/matters-mark-color.svg
apps/web/public/favicon.svg                       (= matters-mark-color.svg)
```

### Refreshing from upstream

When `thematters/design-system` bumps a component or token version,
re-run the scaffold script (assumes both repos are siblings under
`external/`):

```bash
cd ../design-system
node scripts/scaffold-component.mjs Button \
  --to ../matters-studio/apps/web/src/components
node scripts/scaffold-component.mjs TextField \
  --to ../matters-studio/apps/web/src/components
node scripts/scaffold-component.mjs Dialog \
  --to ../matters-studio/apps/web/src/components
node scripts/scaffold-component.mjs Toast \
  --to ../matters-studio/apps/web/src/components

# Refresh tokens
cp packages/tokens/dist/tokens.css \
   ../matters-studio/apps/web/src/styles/tokens.css

# Refresh OG template
cp templates/og-image/template.html \
   ../matters-studio/apps/web/src/og-template/template.html
cp templates/shared/template-base.css \
   ../matters-studio/apps/web/src/og-template/shared/
cp templates/shared/tokens.css \
   ../matters-studio/apps/web/src/og-template/shared/
cp templates/shared/matters-mark-color.svg \
   ../matters-studio/apps/web/src/og-template/shared/
```

Commit the result with a note pointing at the DS version that was the
source. Don't edit the vendored files in-place — fix them upstream and
re-vendor.
